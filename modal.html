1. What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a document as a tree of objects, where each node corresponds to part of the document, such as elements, attributes, and text. This allows scripts (like JavaScript) to dynamically access and manipulate the content, structure, and style of web pages.

How do you select an element from the DOM?

You can select elements from the DOM using various methods provided by the document object in JavaScript. Some common methods include:document.getElementById('id'): Selects an element by its unique ID.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique that involves attaching a single event listener to a parent element instead of multiple listeners to individual child elements. This approach is useful for several reasons: Performance: Reduces memory usage and improves performance, especially in lists with many items. Dynamic Content: Allows you to handle events for elements that are added to the DOM after the initial load without needing to attach new listeners. Simplified Code: Centralizes event handling, making the code easier to manage and maintain.

How do you manipulate an element's attributes and styles using the DOM?

Attributes: To get an attribute: element.getAttribute('attributeName') To set an attribute: element.setAttribute('attributeName', 'value') To remove an attribute: element.removeAttribute('attributeName') Styles: To change inline styles: element.style.propertyName = 'value'; To read a computed style: window.getComputedStyle(element).propertyName These methods allow you to dynamically change how elements appear and behave on a web page.
